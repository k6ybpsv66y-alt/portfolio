PROJET 01 / 03 — WEB / PYTHON

Répertoire Téléphonique

Python HTML CSS
Présentation du projet

Le Répertoire Téléphonique est une application de gestion de contacts développée dans le cadre du cours de programmation Python. L'objectif principal était de concevoir un outil simple, fonctionnel et intuitif permettant à tout utilisateur de centraliser et organiser ses informations de contact en un seul endroit.

Ce projet m'a permis d'explorer les fondamentaux de la programmation Python, notamment la manipulation de structures de données, la lecture/écriture de fichiers, ainsi que la logique métier derrière une application CRUD (Create, Read, Update, Delete). Au-delà du code, ce projet représente ma première approche concrète d'un problème réel résolu par l'informatique.

Fonctionnalités développées
  • Ajout d'un nouveau contact avec nom, prénom et numéro
  • Affichage complet de la liste des contacts
  • Modification d'un contact existant
  • Suppression définitive d'un contact
  • Recherche rapide par nom ou numéro
  • Persistance des données entre les sessions
  • Validation des entrées utilisateur
  • Interface en ligne de commande claire et lisible
Apprentissages clés

Ce projet a été une étape cruciale dans mon apprentissage. J'ai appris à structurer un programme Python en fonctions réutilisables, à gérer les erreurs avec des blocs try/except, et à manipuler des fichiers pour assurer la persistance des données.

Défis rencontrés

Le principal défi a été la gestion de la persistance des données sans base de données formelle. J'ai dû concevoir un système de lecture/écriture en fichier qui reste cohérent même en cas d'interruption du programme.

Python 3 Fichiers .json Programmation fonctionnelle

PROJET 02 / 03 — BASE DE DONNÉES

BD Air Sénégal

MySQL SQL Merise
Présentation du projet

La BD Air Sénégal est un projet de base de données relationnelle développé dans le cadre du cours de systèmes de bases de données. Il simule le système d'information d'une compagnie aérienne, en modélisant les entités fondamentales : les vols, les avions, les passagers, les réservations et les aéroports.

Ce projet couvre l'ensemble du cycle de vie d'un projet de base de données : de la conception du modèle conceptuel (MCD/Merise), à la création physique des tables sous MySQL, jusqu'à l'écriture de requêtes SQL complexes pour interroger et manipuler les données.

Fonctionnalités développées
  • Modélisation conceptuelle avec la méthode Merise (MCD)
  • Création du schéma relationnel normalisé (3NF)
  • Gestion des vols (départ, arrivée, horaires)
  • Gestion des passagers et de leurs réservations
  • Gestion des avions et de leur capacité
  • Requêtes SELECT avec JOIN, GROUP BY, HAVING
  • Insertion, mise à jour et suppression de données
  • Gestion des contraintes d'intégrité référentielle
Apprentissages clés

Ce projet m'a profondément ancré dans la logique relationnelle. J'ai compris pourquoi la normalisation est cruciale pour éviter la redondance des données, et comment les clés étrangères assurent la cohérence entre les tables.

Défis rencontrés

La difficulté principale résidait dans la modélisation des relations entre les entités : un passager peut avoir plusieurs réservations, une réservation concerne un seul vol, mais un vol peut transporter des centaines de passagers. Gérer ces cardinalités correctement dans le MCD a nécessité plusieurs itérations.

MySQL 8 SQL DDL/DML Merise / MCD

PROJET 03 / 03 — EMBARQUÉ / IoT

Poubelle Intelligente via ESP32

ESP32 Arduino IoT
Présentation du projet

La Poubelle Intelligente est un projet de système embarqué réalisé avec un microcontrôleur ESP32. L'idée est simple mais concrète : concevoir une poubelle qui s'ouvre automatiquement lorsqu'une main ou un objet s'en approche, sans contact physique.

Ce projet démontre une maîtrise réelle des systèmes embarqués : câblage électronique, programmation temps réel, gestion des capteurs et des actionneurs. C'est l'un des projets qui m'a le plus passionné car il combine la matière physique et le code de manière très tangible.

Fonctionnalités développées
  • Détection de proximité via capteur ultrasonique HC-SR04
  • Commande d'un servomoteur pour l'ouverture du couvercle
  • Fermeture automatique après délai configurable
  • Gestion des états (ouvert / fermé / en attente)
  • Programmation temps réel sur ESP32
  • Optimisation de la consommation électrique
  • Câblage électronique complet sur breadboard
  • Tests et calibration du capteur de distance
Apprentissages clés

Ce projet m'a initié au monde fascinant des systèmes embarqués. J'ai appris à lire les datasheets des composants électroniques, à comprendre les signaux PWM pour contrôler un servomoteur, et à programmer en C++ sur la plateforme Arduino IDE pour l'ESP32.

Défis rencontrés

La calibration du capteur ultrasonique a été le défi principal. Les valeurs brutes étaient instables et causaient des ouvertures intempestives. J'ai dû implémenter un algorithme de filtrage par moyenne glissante pour lisser les lectures et rendre le comportement stable.

ESP32 C++ / Arduino HC-SR04 Servomoteur SG90
